Top 10 Critical & High Issues

Fix-first list. Each one is load-bearing — they either silently corrupt output (1–5), create maintenance hell (6–7), or break team ergonomics (8–10).

CRITICAL #01

grounding.md is a phantom for existing installs

The file exists in the repo at rules/grounding.md but is missing from the installed ~/.claude/rules/ for users who upgraded from v4.0.x. Every planner, builder, plan-checker, and verifier spawn references @~/.claude/rules/grounding.md and silently gets nothing. The Grounding Protocol and Severity Rubric are improvised. Every 1–5 score is ungrounded.
Where: 8 skills/agents Impact: Every verified phase, every project, silently Fix: install.js post-upgrade routine — diff rules/, copy missing
CRITICAL #02

Builder agent never reads the knowledge base

agents/builder.md has zero references to ~/.claude/knowledge/. Patterns like "Always use lib/supabase/server.ts for mutations" reach the planner but not the agent that writes code. Patterns exist and don't affect output.
Where: agents/builder.md Fix: Add @-reference to knowledge in builder preamble
CRITICAL #03

11 of 14 knowledge files are invisible to every skill

~/.claude/knowledge/ contains qualia-context.md, deployment-map.md, supabase-patterns.md, voice-agent-patterns.md, employees.md, and 6 more — grepping the framework for these filenames returns zero hits. Dead weight. session-digest.md hasn't updated since Apr 12 (the hook that wrote it was deleted).
Impact: Knowledge is write-only; no skill discovers new files Fix: bin/knowledge.js — discovery + 3-tier merge
CRITICAL #04

No per-employee memory scoping

All memory lives in global ~/.claude/knowledge/. Hasan, Fawzi, Moayad share the same files. employees.md is a flat roster, not a scoping mechanism. Personal preferences mix with universal patterns, with no filtering.
Fix: 3-tier architecture — Framework / Employee / Project
CRITICAL #05

qualia-report ERP upload: silent injection & auth failures

Empty API key → sends Authorization: Bearer (blank); 401 handler says "Ask Fawzi" without diagnosing.
SUBMITTED_BY shell-interpolated into node -e → a single quote in git user.name breaks silently.
report_id fallback to empty string on state.js failure → commit message loses ID, ERP payload corrupted.
Where: skills/qualia-report/SKILL.md:84, 114-163 Fix: Guard empty key; env-var pass; validate report_id
HIGH #06

Triple overlap: qualia-optimize / qualia-review / qualia-polish

All three run the same grep patterns (fonts, max-width, gradients, service_role, empty catches, sequential awaits, <img> without next/image). Three output files, three severity scales, one underlying issue. User gets three reports on the same problems.
Fix: qualia-review = detect, qualia-polish = fix, delete qualia-optimize
HIGH #07

4 skills are orphans — never invoked by the Road

qualia-test, qualia-debug, qualia-task, qualia-quick are never auto-triggered by any Road step. The Road has no automated test gate between build and verify. If verification fails, the Road routes to /qualia-plan --gaps, not /qualia-debug.
Fix: Integrate /qualia-test post-build, /qualia-debug on verify-fail
HIGH #08

Color palette drift across 5 surfaces

TEAL_DIM differs rgb(0,130,135) (statusline) vs rgb(0,140,145) (qualia-ui). DIM differs across 3 files. Brand teal in demo.html #00ced9 vs help.html #00ced1. Display fonts split: Fraunces vs Outfit. DIM fails WCAG AA at ~3.2:1 on dark terminals.
Fix: bin/colors.js — single source of truth
HIGH #09

Statusline shows installer name, not current operator

bin/statusline.js:292 reads .qualia-config.json set at install time. On a shared machine, every operator sees the installer's name. Says "Qualia member: Hasan" even when Fawzi is driving.
Fix: Session-scoped identity via git user.name or env var
HIGH #10

/qualia-ship has no state guard and shallow security

No state.js check at start — can invoke from any state. Security check only greps service_role; misses hardcoded keys, tracked .env, dangerouslySetInnerHTML. Post-deploy uses literal {domain} placeholder — expects LLM to hallucinate the URL.
Fix: State precondition + full qualia-review CRITICAL checks + read deployed_url from tracking.json

03

Memory Architecture — 3-Tier Proposal

Current state is flat, untiered, and half-invisible. Proposed architecture gives every finding a single home and makes discovery mechanical, not hardcoded.

1

Framework

Universal · ships with installer

~/.claude/knowledge/framework/
  • stack-patterns.md
  • voice-patterns.md
  • security-patterns.md
  • common-fixes.md
  • KNOWLEDGE-INDEX.md
2

Employee

Per-user · scoped by team code

~/.claude/knowledge/employee/
  • preferences.md
  • my-patterns.md
  • my-fixes.md
  • KNOWLEDGE-INDEX.md
3

Project

Per-repo · version controlled

.planning/knowledge/
  • client-prefs.md
  • project-patterns.md
  • deployment.md
  • KNOWLEDGE-INDEX.md

Discovery utility — bin/knowledge.js

# Merge all 3 tiers — project overrides employee overrides framework
knowledge.js load --type patterns

# Search across tiers, labels source
knowledge.js search "supabase RLS"

# Regenerate INDEX files with last_verified timestamps
knowledge.js index

# Promote a Claude-native memory entry into Qualia structured tier
knowledge.js promote --from employee --to framework

Staleness: INDEX tracks last_verified per file. Entries over 30 days are tagged [STALE] in merge output — skills can warn before using. Conflict: Project overrides Employee overrides Framework. Within a tier, newer timestamp wins. Claude native memory: Keep enabled. Native captures conversational context; Qualia captures structured patterns. Statusline shows both counts: K:14 M:4.

Component Freshness — Fetching Latest

No current mechanism in the framework. Grep for shadcn, radix, component.*fetch returns zero hits. Proposed skill below.

# Pulls latest shadcn components via official CLI
/qualia-components add button card dialog

# Bumps all components to latest; writes .planning/component-changelog.md
/qualia-components update

# Reports N components > X days stale
/qualia-components check

Implementation details

  • Wraps npx shadcn@latest add for shadcn operations
  • Integrates with existing vercel-plugin:shadcn skill
  • WebFetch pulls latest docs (ui.shadcn.com/docs/components/{name})
  • For Radix: npm info @radix-ui/react-{name} version → compares with installed
  • Writes diff summary so planner knows what changed

Integration points

  • Add component-version check to /qualia-verify: flag components > 1 major behind
  • Do NOT auto-update during builds — component breaks should be explicit + versioned
  • Add "component health" panel to statusline for active frontend projects
  • On /qualia-new frontend project, offer to scaffold with latest shadcn

  2. Milestone indicator alongside phase: M2 P3/5 T2/4

PHASE 3 HTML modernization ~1 day
  1. Replace @import with <link rel="preconnect"><link rel="stylesheet"> in help.html

08

Final Verdict

Strengths

The Road skeleton is sound. Context isolation, goal-backward verification, story-file plans, wave-based parallelization, and the state machine hold up against current research (Anthropic, OpenAI, Archon). The harness pattern is genuinely good — these are the right primitives.

Structural debt — three clusters

  • Invisible dependencies — grounding.md, 11 knowledge files, builder↔knowledge disconnect. The framework believes things are loaded that aren't.
  • Silent failures — 15 high-impact silent-fail paths where users get "ok" while operations produced garbage.
  • Orphan skills — 4 commands never touch the Road; 3 triple-overlap. Real mental overhead for the team.
SHIPPING BAR This review would FAIL /qualia-ship on a client project. The framework needs v4.1.1 hotfix before onboarding the next employee, and v4.2.0 before the next major client delivery.

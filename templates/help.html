Road — Core Flow

The seven steps every project walks through.

/qualia-newSet up a new project from scratch — deep questioning, parallel research, REQUIREMENTS.md, ROADMAP.md, approval gate. Use when starting any new client project.
/qualia-planPlan the current phase — spawns planner, validates with plan-checker in a revision loop (max 3), optionally runs discuss/research first. Use when ready to plan a phase.
/qualia-buildExecute the current phase — spawns builder subagents per task with wave-based parallelization. Fresh context per task.
/qualia-verifyGoal-backward verification — checks if the phase ACTUALLY works, not just if tasks completed. Spawns verifier agent.
/qualia-polishDesign and UX pass — anti-AI-slop, genuine craft, responsive, accessible. Run after all phases are verified.
/qualia-shipDeploy to production — quality gates, commit, push, deploy, verify. Use when ready to go live.
/qualia-handoffClient delivery — credentials, handover doc, final update. Use after shipping.

Phase Depth — Optional

Used before / around /qualia-plan when a phase needs deeper context.

/qualia-discussCapture phase decisions, trade-offs, and constraints BEFORE planning. Use for complex phases with regulatory, compliance, or architectural stakes. Creates .planning/phase-{N}-context.md that planner honors as locked input.
/qualia-researchDeep-research a niche domain or library BEFORE planning a specific phase. Spawns the researcher agent with Context7/WebFetch access. Writes to .planning/phase-{N}-research.md.
/qualia-mapMap an existing codebase to infer architecture, stack, conventions, and what's already built. For brownfield projects — run BEFORE /qualia-new so Validated requirements get inferred from existing code.
/qualia-milestoneClose out a completed milestone and prep the next one. Archives the current milestone's artifacts, updates REQUIREMENTS.md to mark v1 requirements Complete, and creates the next milestone roadmap.

Quality

Diagnose, audit, optimize, polish, test.

/qualia-debugStructured debugging — symptom gathering, diagnosis confirmation, root cause analysis. Trigger on 'debug', 'find bug', 'fix error', 'something is broken'.
/qualia-reviewProduction audit with scored diagnostics. Runs real commands, scores findings by severity. Trigger on 'review', 'audit', 'code review', 'security check'.
/qualia-optimizeDeep optimization pass — reads .planning/ AND codebase to find performance, design, UI, backend, and frontend issues. Spawns parallel specialist agents. Supports --perf, --ui, --backend, --alignment, --fix flags.
/qualia-polishDesign and UX pass — anti-AI-slop, genuine craft, responsive, accessible. Run after all phases are verified.
/qualia-testGenerate or run tests for client projects. Trigger on 'write tests', 'add tests', 'test this', 'test coverage'.

Quick Paths

Lightweight alternatives when the full road is overkill.

/qualia-quickFast path for small tasks — bug fixes, tweaks, hot fixes. Skips full phase planning.
/qualia-taskBuild a single task — more structured than /qualia-quick, lighter than /qualia-build. Spawns a fresh builder agent for one focused task.
/qualia-designOne-shot design transformation — critiques, fixes, polishes, hardens, makes responsive. No reports, no choices, just makes it professional.

Knowledge

Persist learnings and log work.

/qualia-learnSave a learning, pattern, fix, or client preference to the knowledge base. Persists across projects and sessions.
/qualia-reportGenerate session report and commit to repo. Mandatory before clock-out.

Session

Hand off and resume context cleanly.

/qualia-pauseSave session context for seamless handoff. Creates .continue-here.md so the next session picks up exactly where you left off.
/qualia-resumeRestore context from a previous session. Reads .continue-here.md or STATE.md, summarizes where you left off, routes to next action.

Navigation

When you don't know what to do next.

/qualiaSmart router — reads project state, classifies your situation, tells you the exact next command. Use whenever you're unsure about your next step.
/qualia-idkDiagnostic intelligence — spawns two isolated scans (planning + codebase) in parallel, cross-references against your confusion, explains the situation in plain language with a concrete next step. Use when something feels off or you need to understand what's going on.
/qualia-helpOpen the Qualia Framework reference guide in the browser. A beautiful themed HTML page with all commands, rules, services, and the road.

Meta

Extend the framework itself.

/qualia-skill-newAuthor a new Qualia skill or agent. Generates the SKILL.md, registers it in the right location, and optionally ships to the framework repo.

Verification Scoring

Every phase is scored on 4 dimensions. Any score below 3 = automatic FAIL.

Correctness

Does it work as expected?

Completeness

Are all requirements met?

Wiring

Is everything connected end-to-end?

Quality

Code quality, security, a11y?

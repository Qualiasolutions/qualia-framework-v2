Commands

Navigation

/qualiaWhat should I do next? Reads project state, tells you the exact command.
/qualia-idkSame as /qualia. For when you're stuck.
/qualia-resumePick up where you left off from a previous session.
/qualia-pauseSave progress. Creates .continue-here.md for next session.

Project Setup

/qualia-newInteractive wizard. Asks about client, scope, design, stack. Creates everything.

Build Loop

/qualia-planPlan the current phase. AI generates tasks with success criteria.
/qualia-buildBuild it. Fresh AI agent per task, parallel waves.
/qualia-verifyVerify it actually works. Greps code, scores on 4 dimensions.

Quick Work

/qualia-quickSkip planning. For bug fixes, tweaks, hot fixes under 1 hour.
/qualia-taskOne focused task. More structured than quick, lighter than build.

Design & Quality

/qualia-designOne-shot design fix. Reads DESIGN.md, kills AI slop, makes it professional.
/qualia-polishFull design pass. Typography, color, states, motion, a11y, responsive, hardening.
/qualia-reviewProduction audit. Security, performance, reliability. Scored diagnostics.

Ship & Deliver

/qualia-shipQuality gates, commit, push, deploy to Vercel, post-deploy verification.
/qualia-handoffClient delivery. Credentials, docs, handover package.
/qualia-reportLog your session. Mandatory before clock-out. Uploads to ERP.

Debugging & Learning

/qualia-debugStructured debugging. Symptom → diagnosis → root cause → fix.
/qualia-learnSave a pattern, fix, or client preference to the knowledge base.
/qualia-testGenerate or run tests. Auto-detects framework.

CLI (Terminal)

qualia-framework installInstall or reinstall the framework.
qualia-framework updateUpdate to the latest version.
qualia-framework versionShow installed version + check for updates.
qualia-framework migrateUpgrade v2 settings to v3.
qualia-framework analyticsHook telemetry, verification pass rates, gap cycles.
qualia-framework teamList, add, or remove team members.
qualia-framework tracesView recent hook activity.

Verification Scoring

Every phase is scored on 4 dimensions. Any score below 3 = automatic FAIL.

Correctness

Does it work as expected?

Completeness

Are all requirements met?

Wiring

Is everything connected end-to-end?

Quality

Code quality, security, a11y?

Rules

  • 1 Feature branches only — never push to main
  • 2 Read before write — understand files before editing
  • 3 MVP first — build what's asked, nothing extra
  • 4 /qualia-report before clock-out — mandatory, enforced by ERP
  • 5 No .env edits — ask Fawzi for secrets
  • 6 Stuck 30+ minutes? Ask Fawzi
